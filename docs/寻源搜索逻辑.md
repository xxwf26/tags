# 寻源搜索逻辑

## 概述

用户上传参考图 + 选择标签（must/fuzzy）+ 选择平台 → 平台搜索召回 → AI 质检 + 打标 → 标签过滤 → 存本地入库。

---

## 一、输入

| 参数 | 来源 | 说明 |
|---|---|---|
| 参考图 | 用户上传 | AI 识别画风标签，可选 |
| 标签 | 用户选择 | 每个标签有 `must`/`fuzzy` 模式 |
| 平台 | 用户选择 | 小红书、微博 |
| fuzzyRatio | 用户设置 | 标签过滤严格度，0.5=命中一半即可，1.0=全部命中 |

标签分 6 个维度：
- **genre（画风）**：日系、厚涂、水墨、古风… → 用作搜索关键词
- technique（技法）、subject（题材）、usage（用途）、tone（色调）、character（人物）→ 仅 AI 过滤用

---

## 二、搜索关键词与过滤标签

```
搜索词 = must 的 genre 标签（没有 must 就用所有 genre 标签）
过滤标签 = 所有选中标签（含 genre 和非 genre）
```

- 搜索词用于平台搜索（XHS 300帖/词，微博 100张/词）
- 过滤标签用于 AI 打标后验证结果是否兼具所有标签

### 标签过滤规则

```
requiredMatchCount = ceil(过滤标签数 × fuzzyRatio)

fuzzyRatio = 1.0 → 必须全部命中
fuzzyRatio = 0.5 → 至少命中一半
fuzzyRatio = 0.0 → 不过滤（全部放行）
```

---

## 三、处理流程（6 张并发）

```
平台搜索 "水墨" → 300 帖
    ↓
┌─────────────────────────────────────────────┐
│ 6 张并发处理：                              │
│                                             │
│ ① sourceUrl 去重                            │
│    已处理过的帖子 → 跳过                     │
│                                             │
│ ② 文字预筛                                 │
│    标题含 cosplay/带货/明星/AI图等 → 跳过    │
│    （免费，不下载不调AI）                    │
│                                             │
│ ③ 下载图片                                 │
│    从平台 CDN 下载到内存                     │
│                                             │
│ ④ hash 去重                                │
│    和库内/本session已有图相似 → 跳过         │
│    （aHash 8×8 灰度指纹，海明距离≤5=重复）  │
│                                             │
│ ⑤ AI 质检（Gemini）                        │
│    is_artwork? quality?（0-10）              │
│    排除：照片/广告/截图/AI生成/cosplay/手办    │
│    quality < 5 → 跳过                       │
│                                             │
│ ⑥ AI 打标（Gemini，同一次调用）              │
│    给图打所有维度的标签                      │
│    （6维 taxonomy 白名单归一）                │
│                                             │
│ ⑦ 标签过滤                                 │
│    AI标签 vs 过滤标签                        │
│    命中数 ≥ requiredMatchCount → 通过         │
│    否则跳过                                 │
│                                             │
│ ⑧ 通过的：                                  │
│    存图片到 /uploads/                       │
│    写 DB（imageUrl=本地路径）                │
│                                             │
│ ⑨ 达到 300 张 → 停止                        │
└─────────────────────────────────────────────┘
    ↓
下一个平台（微博）或结束
```

---

## 四、去重链路（5 层）

| 层 | 时机 | 方法 | 成本 |
|---|---|---|---|
| sourceUrl | 处理前 | URL 字符串比对 | 免费 |
| 文字预筛 | 处理前 | 标题含黑名单关键词 | 免费 |
| hash(库) | 下载后 | aHash 海明距离 ≤ 5 | 低（内存比对） |
| hash(session) | 下载后 | aHash 海明距离 ≤ 5 | 低（内存比对） |
| AI 质检 | 下载后 | Gemini 判 is_artwork + quality | 高（3-5秒/张） |

---

## 五、继续搜索

```
点"继续搜索更多"
    ↓
用非 genre 标签做搜索词（找不同帖子）
    ↓
预加载本 session 已有结果的 URL + hash
    ↓
同样流程（去重不重复）
    ↓
结果加到同一 session（不新建记录）
```

## 六、关键参数

| 参数 | 当前值 | 作用 |
|---|---|---|
| XHS 召回 | 300 帖/词 | 每个关键词搜 300 帖 |
| 微博召回 | 100 张/词 | 每个关键词搜 100 张 |
| 并发数 | 6 | 6 张同时调 AI |
| AI 质检 | Gemini 单模型 | is_artwork + quality + tags（一次调用三合一） |
| MIN_QUALITY | 5 | quality < 5 的跳过 |
| fuzzyRatio | 0.5（前端默认） | 至少命中一半过滤标签 |
| targetResults | 300 | 达到 300 张结果就停 |
| 同帖去重 | noteId/sourceUrl | 微博同帖只留第一张 |
| 图片存储 | /uploads/search-{sid}-{idx}.jpg | 本地保存，CDN 过期也能看 |
| 文字黑名单 | ~100 个词 | cosplay/带货/明星/AI图等 | 
| AI 排除 | AI生成图/cosplay/手办/照片/广告 | GATE_PROMPT 明确排除 |
| 子进程 | execFile | chromium 不崩主服务 |

## 七、可优化方向

### 1. fuzzyRatio 默认 0.5 太松
- 当前默认 0.5 = 命中一半标签就放行
- 改为 1.0 = 必须全部命中才放行
- **影响**：结果更精准，但数量减少
- **建议**：默认 1.0，用户想更多可手动调低 | 前端滑块已有，改默认值即可

### 2. 搜索词只取 genre 标签，非 genre 标签无法搜索
- 选"场景"(subject) 不参与搜索，只能 AI 过滤 → 召回的 300 帖大部分不含"场景" → 70% AI 调用浪费
- **优化**：搜索词扩展到所有标签（逐个搜，结果合并去重）
- **代价**：搜索量 ×N（但并发处理不变，只是召回阶段多调几次平台搜索）
- **效果**：召回更精准，AI 调用更少

### 3. AI 质检 + 打标一次调用三合一
- 当前 callGemini 一次同时做 is_artwork + quality + tags
- 优点：一次调用完成所有判断
- 缺点：prompt 长（含 43 个标签词表），响应慢
- **优化**：分两步——先用短 prompt 做 is_artwork + quality（快），通过后再用长 prompt 打标（只对通过的调）→ 省掉 50%+ 的长 prompt 调用
- **代价**：通过的图要调两次 AI（但总量减少）
- **效果**：非绘画帖只调短 prompt（1秒），不浪费长 prompt（3-5秒）

### 4. 文字黑名单覆盖面
- 当前 ~100 个词，偏 lifestyle/购物
- 可补充：摄影、新闻、体育、政治、社会等
- **影响**：进一步减少无效 AI 调用

### 5. 小红书搜索帖子类型过滤
- 当前不区分视频/图文帖 → 视频帖下载图可能失败
- **优化**：XHS 搜索时跳过 type=video
- **效果**：减少失败下载

### 6. 微博同帖多图只取第一张
- 当前微博同一帖子(noteId)只保留第一张图
- **优化**：保留最好的那张（最大/最清晰）而非第一张
- **影响**：结果质量略提升

### 7. 图片存储去重
- 当前每张都存到 /uploads/，即使内容相同也存新文件
- **优化**：hash 相同的图复用已存文件
- **影响**：省磁盘空间

### 8. 标签过滤用精确匹配
- 当前 checkTagFilter 用 label 精确匹配
- **问题**：AI 打标和用户选择可能不完全对齐（如"厚涂"vs"厚涂技法"）
- **优化**：用 tagId 匹配而非 label，或加别名

### 9. 继续搜索的关键词
- 当前用非 genre 标签做关键词
- **问题**：如果用户只选了 genre 标签，继续搜索没有非 genre 词可用
- **优化**：继续搜索用第二个 genre 标签（第一个已搜过）

### 10. 进度显示
- 当前每批（6张）更新一次进度
- **问题**：如果第一批处理很快（AI 快速拒绝），前端可能来不及显示进度
- **优化**：开始处理前先更新一次"0/300 处理中"

## 八、文件结构

```
search.service.ts     → 主逻辑（搜索+质检+过滤+入库）
search.controller.ts  → 端点（start/abort/continue/sessions/results）
xhs-search.mjs       → 小红书搜索子进程
weibo-search.mts     → 微博搜索子进程
xhs-login.mts        → 小红书扫码登录子进程
mhs-search.mts       → 米画师搜索子进程
mhs-tags.mts         → 米画师标签拉取子进程
settings.controller  → cookie/扫码登录端点
image-proxy         → 外站图片代理（防盗链）
promote-helpers.ts  → 入库去重共用
clip.ts             → CLIP 视觉相似度（默认关）
ai.ts               → Gemini 质检 prompt + 打标 prompt
```
