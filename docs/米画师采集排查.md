# 米画师采集排查指南

> 用途：合并代码后「米画师爬不到作品」时，按本文档自查。
> 相关代码：`server/src/modules/crawl/mihuashi.ts`

## 一、我们怎么爬米画师（原理）

不是逆向签名，而是**用 playwright 开无头浏览器模拟真人操作**：

1. 无头 Chromium 打开 `mihuashi.com/artworks` 列表页；
2. 在页面上**点击画风标签按钮**（如「日系」「厚涂」）；
3. 滚动加载，同时**拦截页面发出的 `/api/v1/artworks/search` 接口响应**，从中收集作品图 URL；
4. 画师主页采集（`fetchMihuashiArtistWorks`）另需登录态文件。

### ⚠️ 关键反爬点（代码已内置，切勿在合并时删掉）

米画师会检测 playwright 特征，识破后返回**假的「签名错误」403**来迷惑你——真实签名其实是对的。破解靠两行，都在 `mihuashi.ts`：

- 启动参数：`--disable-blink-features=AutomationControlled`
- context 注入：`addInitScript(() => Object.defineProperty(navigator,'webdriver',{get:()=>undefined}))`

合并代码若遇冲突，务必保留这两行，否则 search 接口会返回 `{"error":"signature invalid","msg":"签名错误"}` 403。

## 二、爬不到的常见原因（按概率排序）

### ① playwright 浏览器内核没装 ⭐ 最可能

`npm install` 只装 playwright 的 npm 包，**不会自动下载 Chromium 内核**，本项目也没配 postinstall 自动装。缺内核时一跑就报 `Executable doesn't exist at ...`。

```bash
cd server && npx playwright install chromium
```

内核会装到 `%LOCALAPPDATA%\ms-playwright\`（如 `chromium-1228`）。

### ② 关键词和米画师官方标签对不上

`searchMihuashi` 靠**点击页面上「文字 = 关键词」的按钮**。如果传入的画风词（如「日系插画」）在米画师页面上找不到对应按钮（页面只有「日系」），会**直接返回空、不报错**。

米画师只认它自己那套官方标签。目前已知的两个维度：

- **画风（19 个）**：日系、平涂、萌系、厚涂、赛璐璐、古风、中国风、童趣、写实系、韩系、少女漫画、欧美系、水彩、美式卡通、白描、科幻风、像素风、水墨、硬派
- **类型（24 个）**：头像、插图、Q版、自设/OC、立绘、角色设计、壁纸、封面、场景、海报、概念设计、印花、图标、Live2D、CG、和纸胶带、像素图、卡牌、条漫、UI、版型、分镜、抱枕、特效

搜索词必须是这些**原词**（「日系」而非「日系插画」）。

### ③ 画师主页采集缺登录态文件

`fetchMihuashiArtistWorks`（按画师主页抓作品）需要 `server/mhs-auth.json` 登录态。**该文件在 .gitignore 里、不随代码同步**，所以你那边没有 → 画师主页采集必然失败。

- 注意：**按画风标签搜作品（`searchMihuashi`）不需要登录**，这条路应该能用。
- 若要用画师主页采集，需自己用 playwright 登录一次米画师、把 storageState 存成 `server/mhs-auth.json`。

### ④ IP / 风控 / 网络

米画师对异常访问有风控。短时间多次快速跑、或 IP 被限，也会 403。偶发的话隔一会儿再试。

## 三、自查顺序

```bash
# 1. 先装浏览器内核（最可能的原因）
cd server && npx playwright install chromium
```

1. 装完内核重试 → 大概率解决；
2. 仍返回空 → 确认搜的关键词是上面列的米画师官方标签**原词**；
3. 报 `signature invalid / 签名错误` 403 → 说明反检测被绕过了，检查本地 `mihuashi.ts` 那两行 stealth 有没有被合并冲突弄丢（见第一节）；
4. 只有画师主页采集失败、按标签搜正常 → 是缺 `mhs-auth.json`，属正常，需自己登录生成。
